Latex

昨天踩了一天的坑，学习如何优雅的使用Latex写编译原理的作业，总结一下。主要是基于Mac OS。

下载


推荐下载Mac tex(4G)、这个套件里面有排版编译器和一个编辑器，但是这个编辑器不太好用，推荐使用TexPad，这个需要下载破解版。但是网上版本不一致，需要尝试。这样基本就算完成了，我好像也没有配置什么环境，网上还有一种是sublime+skim的模式，试了一下弃用了。

使用


这个Latex其实是不太好用，但是好在编出的论文真的好看，而且自己写了模版就方便多了。有几个注意事项。

  • 切换pdftex到xelatex编译器

  • 有一些标红的奇奇怪怪的错不用管，双击排版强行生成pdf

  • 注意一些特殊符号%、&&等等好多需要转义

  • 一些希腊字母需要用它自带的，不能自己打

  • 插入图片很麻烦，这个我也没搞懂🐷

  • 一句话太长需要手动换行

  • 换行和空格都需要特殊字符来打


    • 推荐两个经得住考验的博客：

    http://www.hanlindong.com/2017/latex-on-mac/

    https://www.jianshu.com/p/689673271df1

    今天是开学第一天，上了六节大课，但是我没怎么听进去，以后不能这样了

    内存中的堆(stack)和栈(heap)


    内存中的堆和栈与数据结构中的堆和栈没有任何关系

    Stack memory内存空间由操作系统自动分配和释放，Heap Memory内存空间手动申请和释放的，Heap Memory内存常用new关键字来分配。

    Stack Memory内存空间有限，Heap Memor的空间是很大的自由区几乎没有空间限制。

    内存中几个重要的指针


    bp：基址寄存器(stack pointer)，一般在函数中用来保存进入函数时的sp的栈顶基址

    sp：指向栈顶的指针

    cs：中保存了代码段的基地址

    ds：保存的是数据段的基地址

    ip：保存的是所要执行的下一条指令的地址。

    2019-06-16 计算机组成原理

    存储体系😐


    访存瓶颈：大容量和高速度 -> 多级存储体系

    命中时间：访问存储器层次中的高层存储器所需要的时间，包括了判断当前访问是命中还是缺失所需的时间

    缺失代价：将相应的块从低层存储器替换到高层存储器中，以及将该信息块传送给处理器的时间之和

    存储器技术

    • rom:只读，断电保留

    • ram:随机存取，断电不保留

      • 主存（DRAM）：不能长久地保持数据，必须周期性地刷新
        （每次刷新一行）多bank并行（提高带宽）

      • cache（SRAM）:不需要刷新，并且其访问时间与周期时间非常相近

      • 闪存：这种非易失存储器用作个人移动设备中的二级存储器，电可擦除的可编程只读存储器

      • 磁盘（机械器件）：访问最慢，容量最大，顺序存储器

        • cache-增收


        block的大小：增加块的大小通常会引起缺失率的降低，但是当块大小增加到一定程度时，缺失率也会增加，这是因为块的竞争增加。另外，仅仅增加块大小会使缺失成本增加

        相联度

        替换算法：LRU,随机替换

        写一致性：直写，回写，写分配

        多级cache:一级cache致力于减少命中时间获得较短的时间周期或者较少的流水线，二级cache则主要针对改善缺失率以减少长时间的访存代价

        虚存-止损(百万cycle太长)


        Page(4k)

        相联度：全相联 写一致性：回写法

        TLB缺失：使用软件解决，MIPS支持随机替换
