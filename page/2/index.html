    SQL is Bag language

    • 选择、投影、join等操作可以有相同的行

    • Bag Union:An element appears in the union of two bags the sum of the number of times it appears in each bag

    • Bag Intersection:An element appears in the intersection of two bags the minimum of the number of times it appears in either

    • Bag Difference:An element appears in the difference A – B of bags as many times as it appears in A, minus the number of times it appears in B.(But never less than 0 times.)


      • 有些对于集合（Set）的操作不适用于Bag

      根据违反BCNF的函数依赖拆表：Replace R by relations with schemas:

      图中的最短路径


      ⚠️时间复杂度可以改进到O(mn)时间

      空间复杂度的改进：我们不对每个值i记录M[i,v],而是对每个 结点v更新一个值M[v]，M[v]=min(M[v],min(cvw +M[w]))（多加一条边） 这样空间复杂度改进到O(n)

      复原最短路径：记录每个结点通向终点t的路径上 它后面的第一个结点；只要距离M[v]更新，我们就更新这个值（first[v]）

      如果指针图P包含一个圈C，那么这个圈一定有负费用

      算法终止信号：某一次迭代中所有的M[v]值都保持不变

      图中的负圈


      n-1步还停不下来，就说明有一个负圈

      存在O(mn)时间的算法找到一个负圈（增加一个终点）

      网络流

      最大流问题


      问题描述：有向图G,存在单一源点和单一汇点，每条边关联一个容量，非负数C

      解决思路：引出原始边（前向边）和剩余边（后向边）的概念；bottleneck(P,f)是P上任何边关于流f的 最小剩余容量；把剩余图中的任何一条s-t路径认为是一条增广路径
      算法：Ford-Fulkerson算法

      算法分析：因为有一个上界，我们知道 Ford-Fulkerson算法 一定会终止,算法时间复杂度：O(mC)

      最大流和最小割


      最大流的值等于最小割

      选择好的增广路径


      算法时间复杂度O(mC)，有些时候算法的执行会非常没有效率（因为选择了一条瓶颈容量很小的增广路径，导致收敛很慢）

      算法改进：选择具有大的瓶颈容量的路径，那么算法进展会更快些

      算法分析：外层While循环的迭代次数至多是 1 + ⎡log2 C⎤， ∆每次缩小一半

      二分匹配问题


      将s连到L中的每个点，并赋流值为1；R中的每个点连到t,并赋流值为1;图中的最大s-t流的值等于G中最大匹配的大小

      NP与计算的难解性


      可行性的定义：存在多项式时间的算法

      定义X至少和Y一样难：如果对于问题Y，能够用多项式个标准的计算步骤+多项式次调用解问题X的黑盒子来解问题Y。那么记作 Y <= X;”Y多项式时间可归约到X“或”X至少像Y一样难“

      多项式时间归约


      几个基本问题：独立集问题，顶点覆盖问题（S是一个独立集当且仅当它的补V-S是一个顶点覆盖）

      集合覆盖：试图用一组较小的集合覆盖一个任意的对象集合（顶点覆盖<=集合覆盖）

      SAT问题：是否存在满足的真值赋值（3-SAY<=独立集）

      3-SAT <= 独立集 <= 顶点覆盖 <= 集合覆盖

      自归约问题:寻找问题 <= 判别问题

      P类问题      ：存在多项式时间解法的问题的集合P(判定素数的问题：AKS算法;X和Y有没有公因数;编辑距离;高斯解方程)

      NP：是所有存在有效验证程序的问题的集合

      NP完全:(电路可满足性)如果Y是一个NP完全问题，X属于NP且Y<=X，则X是NP完全的

      一般经验是：NP问题要么是P，要么是NP完全

      多对一关系：at most 1 箭头指向的一端是“唯一的”

      圆箭头：有且只有一个

      多端关系转换成双端：关系变实体，注意多对一的箭头

      子类：1、三角形的尖指向父类 2、每种实体都是包含根节点的一棵子树 3、只有根有主键，并且充当所有实体的主键

      模型约束：实体完整性（若属性A是基本关系R的主属性，则属性A不能取空值）、参照完整性（外键或者取参照关系中的某个主键值，或者取空值（表示还没有分配）

      计数逆序


      穷举：Θ(n2) 排序:Θ(nlgn)

      算法：O(nlogn)

      Merge-and-Count(A,B) 维护一个Current指针指向每个表，初始化指向首元素
维护一个变量Count用于逆序的计数，初始为0 
While两个表都不空
令ai与bj是由Current指针指向的元素 把这两个表中较小的元素加到输出表中 
If bj是较小的元素 then
把Count加上在A中剩余的元素数 
Endif
把较小元素选出的表中的Current指针前移
Endwhile
一旦一个表为空，把另一个表剩余的所有元素加到输出中 返回Count和合并后的表

Sort-and-Count(L) 
If这个表中有一个元素，
then
    没有逆序
Else
    把这个表划分成两半， A包含前⎡n/2⎤ 个元素
    ⎣n/2⎦ B包含剩下的 个元素
    (rA,A)=Sort-and-Count(A) (rB,B)=Sort-and-Count(B)
    (r ,L)=Merge-and-Count(A,B)
Endif 
返回r=rA+rB+r以及排好序的表L

      最邻近点对


      算法：O(nlog2n)

      Closest-Pair(p1, ..., pn) {
Compute separation line L such that half the points are on one side and half on the other side.

δ1 = Closest-Pair(left half) 
δ2 = Closest-Pair(right half) 
δ = min(δ1, δ2)
Delete all points further than δ from separation line L O(n)
Sort remaining points by y-coordinate.
Scan points in y-order and compare distance between each point and next 11 neighbors. If any of these distances is less than δ, update δ.
return δ. 
}

      贪心算法


      直观上说，一个算法是贪心的，如果此算法是通过一些小的步骤来建立一个解，在每一步根据局部情况选择一个决定使得某些主要的指标能得到优化。存在一个局部判断规则可以用来构造问题的最优解。

      两个基本的方法来证明一个贪心算法对一个问题能够提供一个最优解

      1、贪心算法领先：每一步都比其他的算法好，从而证明产生了一个最优解

      2、交换论证：考虑对这个问题的任何可能解，逐渐把它转换成由贪心算法找到的解，而且不影响解的质量。从而证明了贪心法找到了一个至少与其他解一样好的解。

      图

      基本定义和应用


      图的表示：

      领接矩阵：顶点之间有边就标1，主对角线是0（自己到自己是0）

      领接链表：把与它有边相连的顶点接到一个链表上

      定理：设G是一个具有n个结点的无向图。下面任意两条都可以推出第三条：1、G是连通的；2、G不包含一个圈；3、G有n-1条边

      图的连通性与图的遍历


      BFS：O(m+n)可以生成层 DFS：O(m+n)

      二部图(红蓝着色)


      定理：图G是二部图当且仅当图中没有奇圈

      定理：设G是一个连通图,L0, …, Lk 是从顶点s由BFS 算法生成的层。那么下面两件事一定恰好成立其一:
      i. G中没有边与同一层的两个结点相交。这种情况下G 是二部图，其中偶数层的结点可以着红色，奇数层结 点可以着蓝色。
      ii. G中有一条边与同一层的两个结点相交。此种情形下， 存在一个奇圈，不可能是二部图。

      强连通性


      如果有向图中每对顶点是相互可达的，那么此图是强连通的

      存在O(m + n)的有效算法判别图G是否强连通

      Pick any node s.
      Run BFS from s in G.
      Run BFS from s in Grev.
      Return true iff all nodes reached in both BFS executions.

      有向无圈图(DAG)和拓扑排序

      To compute a topological ordering of G:
Find a node v with no incoming edges and order it first
Delete v from G
Recursively compute a topological ordering of G-{v} and append this order after v

      上面算法在 O(m + n) 时间内找到一个拓扑排序.
