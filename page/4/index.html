含有事务应用的删除操作（多表）


事务就是一大串操作要不全执行，要不一个也不执行，执行失败时会回滚到开始的状态

今天在实现的时候尝试了很多种方法，举一个🌰
算了忘了，yii框架里面有一些封装好的findOne()、findMany()方法，但是不是很好用。还有一种是需要在控制器和模型中都实现的方法，尝试了一下也没有成功。最终!!直接使用SQL语句，这时需要yii提供的PDO，具体实现如下：

public function actionDelete($id)
{
    //$this->findModel($id)->delete();
    $transaction = Yii::$app->db->beginTransaction();
    try{
        //删除评论
        $sql = "delete from comment
        WHERE userid=$id";
        $connection=Yii::$app->db;
        $command=$connection->createCommand($sql);
        $comment=$command->execute();

        if(!$comment)
            throw new \Exception('操作失败！222');

        //删除文章
        $sql = "delete from post
        WHERE author_id=$id";
        $connection=Yii::$app->db;
        $command=$connection->createCommand($sql);
        $post=$command->execute();

        if(!$post)
            throw new \Exception('操作失败！111');

        //删除用户
        $sql = "delete from user
        WHERE id=$id";
        $connection=Yii::$app->db;
        $command=$connection->createCommand($sql);
        $user=$command->execute();

        if(!$user)
            throw new \Exception('操作失败！000');

        //以上执行都成功，则对数据库进行实际执行
        $transaction->commit(); 
        // return $this->refresh();
    }catch (\Exception $e){
        //如果抛出错误则进入catch，先callback，然后捕获错误，返回错误
        $transaction->rollBack();
        return $e->getMessage();
    }
    return $this->redirect(['index']);
}

其中包括实现事务（Transation）的结构和加入SQL语句的操作

触发器控制下的添加操作


mysql触发器是在事件（进行插入、更新、删除操作前后）发生时触发的动作，我先写一个简单的触发器，这个就是实现触发器的基本格式

DROP TRIGGER IF EXISTS `tri_post`;
create trigger tri_post
before insert on post for each row
begin
     if LENGTH(new.content)>10 then 
             insert into post (title, content, tags, status) 
        values( new.title, new.content, new.tags, new.status); 
    end if;
end;

2019-05-13 Android

Service


和Activity很像，区别在于Service没有界面

onCreate()方法：当service被多次启动时，真正运行的service只有一个

onStartCommand()方法：会生成一个notification

onDestory()方法：关闭service

STICKY:当android程序被无缘无故kill掉时，设置属性STICKY的程序会自动重启

将service绑定到activity：onBind()方法

在主activity的onServiceConnected()方法会被回调,另外，activity中还有onServiceDisconnected()方法，可以把一些善后工作放在这个方法中执行

创建前台Service(很少见)：在创建到前台中时会有Notification

因为activity和service都是运行在主线程中的，应当避免在onStartCommand()方法中执行费时的操作。当service要处理较为耗时的操作时，使用IntentService，它是单独一个线程，使用时要重写onHandleIntent()方法

获取系统服务：getSystemService() 闹铃服务(AlarmManager)
补充：PendingIntent是Intent的进一步封装

Activity之间的通信


startActicityForResult方法（intent,requstCode）其中requstCode用来帮助判断是调用者的哪个请求

onActicityResult方法：被调用的ACticity finish()之后会回调调用者的这个方法

Intent.putExtra方法中可以放好多key-value

resultCode:可以帮助调用者判断这是第几次被调用

Android 广播(Broadcast和BroadcaseReceiver)

自定义广播


一开始所有的BroadcaseReceiver都是处于休眠状态，当有一个信号时，操作系统会遍历BroadcaseReceiver的链表，查看谁关心就叫醒他

广播：Context.sendBroadcast()、sendOrderBroadcast()、sendStickyBroadcast()方法把Intent对象以广播的形式发送出去，双方都可以有权限限制

Receiver:需要注册(在注册时可以设置action)，onReceive()方法进行接收

需要通过intent发送，其中有action用来过滤（匹配）

接收广播不需要界面

注册方法分为静态注册和动态注册，主要区别是：静态注册时，即使APP从来没有被执行过，onReceive方法依然会被执行；动态注册时，当Context对象被销毁时，Receiver也随之被销毁(也可以设置过滤等操作)

接受系统广播（电量低、时区改变等）

Service :后台运行（例如：下载音乐）和跨进程访问
Context.startService()和Context.bindService()

Service不能自己启动，需要Activity使用startService()方法（只能调用stopService()方法结束）。而使用bindService（）方法启用服务，调用者与服务绑定在了一起。（使用onUnbind方法解除绑定）

创建Service:onCreate()方法 可以设置使用权限
